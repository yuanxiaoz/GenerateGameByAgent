# NPC关系网络系统使用指南

## 核心概念

NPC关系网络让游戏世界更真实：
- NPC之间有独立的关系（师徒/道侣/仇敌/盟友等）
- 关系影响NPC行为和剧情发展
- 玩家可以利用或影响NPC关系

## 使用场景示例

### 场景1：宗门内斗
```
张长老（保守派）vs 李长老（改革派）
- 关系：竞争，score: -40
- 玩家帮助李长老 → 张长老对玩家好感-10
- 两派弟子也会相互敌视
```

### 场景2：三角恋
```
王师兄暗恋林师姐（单向，score: 70）
林师姐是玩家道侣（双向，score: 90）
→ 王师兄嫉妒玩家，可能暗中使坏
```

### 场景3：仇敌的仇敌
```
玩家仇敌：赵魔头
赵魔头仇敌：孙剑仙
→ 玩家可以联合孙剑仙对付赵魔头
```

### 场景4：师门关系链
```
掌门 → 大长老（师徒）→ 核心弟子（师徒）
玩家得罪核心弟子 → 大长老不满 → 掌门态度转冷
```

## AI提示词集成

在 `businessRules.ts` 中的 NPC_RULES 添加：

```typescript
[NPC关系网络]
NPC间关系:读取社交.关系矩阵.edges,筛选相关NPC
关系影响行为:仇敌见面→敌视/攻击;盟友求助→优先帮助;师徒→服从/保护
玩家影响:帮助NPC的仇敌→该NPC好感下降;介绍NPC认识→可能建立新关系
关系推演:NPC互动后更新score;重大事件±30~50;记录events
```

## 实现要点

1. **关系初始化**：创建NPC时，根据身份/宗门/背景自动生成关系
2. **关系更新**：每次NPC互动/重大事件后更新
3. **关系查询**：AI需要时查询NPC关系，影响剧情走向
4. **连锁反应**：一个关系变化可能影响多个NPC

## 优势

- **世界真实感**：NPC不再是工具人，有自己的社交网络
- **剧情深度**：复杂的人际关系带来更多剧情可能
- **策略性**：玩家可以利用关系网达成目标
- **连锁反应**：一个行动影响多个NPC，增加决策权重
